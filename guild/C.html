# 2. 计算机硬件

  1. 主机部分

    1. CPU
      1. 运算器(数值运算、逻辑运算)
      2. 控制器(控制设备)，比如 word 进行打印.
    2. 内存储器
      1. 只读存储器(ROM, read only memory), BIOS.
      2. 随机访问存储器(RAM, random access memory), 我们平时所说的内存. 内存中的数据可以随意访问.
        1. 内存中靠电信号来存储用户数据.
        2. 缺点：一旦断电，数据丢失. 不能持久化保存数据.
        3. 优点: 从内存中读写数据速度非常高.

  2. 外设部分

    在计算中，输入和输出相对于计算机而言的.

    1. 输入设备
      1. 输入相对于计算机而言，读操作.
      2. 键盘、鼠标、摄像头、麦克风等等
    2. 输出设备
      1. 输出相对于计算机而言，写操作.
      2. 显示器、投影仪、打印机等等
    3. 外存储器
      1. 通常所指就是硬盘.
      2. 一般来讲，硬盘靠磁信号来存储数据.
      3. 优点: 能够持久化保存数据.
      4. 缺点: 硬盘相对于内存而言，读写效率非常低.

  3. CPU-内存-硬盘在程序中的作用

  4. 程序在运行过程中会处理一些数据. 由我们的 CPU 来处理数据(计算数据).

  5. 如果程序中的数据在硬盘中存储，CPU 直接和硬盘进行交互.

    1. CPU 运行效率非常快.
    2. 硬盘读写数据的效率又非常低.
    3. CPU很快，读取数据时，写入数据时就会需要等待. 在这个等待时间内，CPU啥也干不了，资源浪费.
    4. 如何解决矛盾？
      1. 就是在CPU和硬盘之间，增加中间的硬件，内存.
      2. CPU直接交互的对象从硬盘变为内存.
      3. 当CPU要处理数据时，首先就需要将硬盘中的数据加载到内存中. CPU在对内存中的数据进行计算. 当计算完毕，需要持久化保存数据时，再将内存中的数据保存到硬盘中.
    5. 从程序角度，内存就是用于存储程序运行过程所需要的临时数据. 硬盘的作用作用就是用于存储程序运行过程中需要长久保存的数据.

# 3. 计算机软件

  1. 计算机操作系统

    1. 提问: 当我们购买一台电脑的时候，如果没有安装操作系统(裸机)，我们能不能让电脑播放音乐?

    2. 是可以的. 不过就是比较麻烦.

    3. 计算机能不能播放音乐，取决于声卡有没有工作. 只要编写程序，让声卡进行工作，将电脑存储的音乐文件(010101010101010), 声卡就可以将这些0101等的数字信号转换为声音信号，进而将声音信号发给音箱，此时音乐就播放出来.

    4. 只需要我们自己编写程序让显卡工作，交给显示器显示.

    5. 不同的电脑使用的硬件可能有所差异，意味着我们可能需要针对不同的硬件编写程序，使得该硬件工作.

    6. 假设，我们要开发一个播放器软件. 先要对底层硬件编写程序，再开发播放器软件.

    7. 所以，有一些非常优秀公司、商业团体、开源组织、个人就编写出一套针对硬件的底层程序. 显卡、网卡、声卡等等，把这些程序打包好，变成我们的操作系统程序.

    8. 至此，如果我们再想开发一个播放器软件，只需要调用操作系统编写好的底层程序，完成我们的业务即可.

    9. 操作系统的作用：

      1. 对下，管理计算机的硬件资源.
      2. 对上，提供使用计算机资源的操作方式.
        1. 系统调用：一套已经写好的代码. 代码针对硬件进行工作.
        2. 终端命令：针对开发人员、普通用户. 开发人员使用较多.
        3. 图形界面：普通用户.
      3. 常见操作系统：
        1. Windows 操作系统：提供终端命令、图形界面、系统调用. 微软公司.
        2. Mac 操作系统：提供终端命令、系统调用、图形界面. 苹果公司.
        3. Linux 系统操作：提供终端命令、系统调用、图形界面.

# 4. 编译器和编程语言

  1. 对于开发人员：重点学习终端命令、程序编写. 其中终端命令在后期的 Linux 课程中会讲解. 我们重点学习的内容是通过程序来操作硬件.

  2. 需要我们掌握一门编程语言. 常见编程语言有那些？

    1. C/C++语言、Java语言、PHP语言、Python语言等，无论什么语言都是和计算机交流的. 我们学习C/C++.

    2. 编程语言到底是什么东西？

      1. 作用：用于控制计算机.
      2. 组成：文字和符号组成的.
      3. 学习: 学习文字和符号编写的规则.

  3. CPU 只认识二进制的指令. 我们编写的程序叫做源代码. 源代码人都能看得懂，但是机器是看不懂. 所以我们得想办法让计算机能够识别我们编写的源代码.

    1. 编译器. 一个运行在操作系统之上的程序.
    2. 我们只需要将我们编写的源代码给这个编译器程序，编译就会将我们所编写的源代码转换成机器能够识别的二进制指令.

  4. 如果你用 Java 语言编写了程序，编写的程序也是不能够直接运行。需要 Java 语言的编译器将 Java 语言编译成机器能够识别的指令.

  5. 如果你用 Python 语言编写了程序，编写程序同样不能够直接运行。需要 Python 的编译器程序负责将 Python 程序编译成机器能够识别指令.

我们无论学习任何一门语言，想要把程序运行起来，那么就需要至少两件事：

  1. 学习该种语言的文字和符号编写的规则. 语法规则.
  2. 需要在操作系统上安装对应语言的编译器程序.

# 5. C语言标准和发展史

  1. 语言发展史

    1. 机器语言. 01010101 01010101 01110101 01010101

    2. 汇编语言. 助记符. 01010101 取名字 abc， abc 代表 01010101.

      汇编编译器：负责助记符转换为具体的二进制指令.

      1. 硬件不同所支持的指令就不同. 指令集.
      2. A CPU 、 B CPU
      3. 使用汇编语言针对硬件编程.
      4. 程序跨操作系统就很差. 跨平台性比较差.

    3. 高级语言

      1. C 语言：当使用C语言编写完程序之后，安装相应的编译器，编译器就会将我们编写的C代码转换成对应平台的代码.
      2. C++语言
      3. Java 语言、OC语言、C#语言(C sharp)、Python语言。
      4. 机器生汇编、汇编生B、B生C、C生万物.

    4. 从语言的发展历程：1. 语言的使用越来越简单. 2. 学习成本越来越高(越高级的语言，隐藏的细节就会越多.).

  2. C语言标准

    C语言是1972年在贝尔实验室诞生的. 由丹尼斯·里奇开发. 参考了托马森的B语言.

    1970-80年代，C语言被广泛应用，产生很多不同的C语言版本. 程序可移植性比较差.

    1983年，美国国家标准委员会(ANSI) 成立一个小组来制定C语言的标准. C语言支持哪些语法、支持哪些功能等等.

    1989年，通过了C语言的第一个标准. C89标准.

    1990年，国际标准化组织(ISO) 和 国际电工委员会(IEC) 将 C89标准当做国际的C语言标准. C90标准. C89和C90指的是同一个标准.

    1994年 ISO和IEC 对 C89标准进行修订. C94标准. 由于并没有增加新的语法特性，还是叫做 C89或者C90.

    1995年 ISO和IEC再次做了修正，C95 标准.

    1999年 ISO 和 IEC 发布了C语言新标准. C语言第二个标准. 在该标准中，新增许多实用的C语言语法特性. 增加新的关键字、可变长数组等等.

    2007年，重新修订了C语言.

    2011年， 发布新的版本。新增了一些语法，泛型、国际化支持. 到目前为止最新的版本是 C11.

  3. 如何去理解标准，标准对我们编写程序有什么影响呢？

    1. C语言的标准我们理解为是一本C语言说明书. 并没有强制性的约束力. 微软拿到了说明书，微软认为标准里面有些标准不合理，他不支持这个特性. 微软认为某些特性非常好，但是标准里面没有，微软自己新增语法.
    2. 如果编译器不支持的话，我们使用这些语法仍然会报错的.
    3. 编译器的版本也会影响程序. 编写程序之前要确定编译器版本.
    4. 介绍下常见的C/C++编译器:
      1. Borland C++ 宝蓝公司
      2. Intel C++ 英特尔编译器
      3. VC++ 微软公司
      4. g++ 编译器（GCC, 编译套件）, Linux 操作系统使用就是该编译器. 对标准支持最好.

# 6. C语言优点和应用领域

  1. 优点：

    1. 执行速度快. 对于很多语言， Python语言，里面的涉及高效率的程序部分，就是用C语言编写的.
    2. 功能强大. C语言可以和硬件、操作系统打交道.

  2. 缺点：

  3. 使用C语言开发项目周期比较长.

  4. 编码自由, 经验不足容易出错.

  5. C语言的应用领域：

    1. 广泛.
    2. Web 网站应用.
    3. 游戏开发.
    4. 嵌入式(智能硬件).
    5. 操作系统.
    6. .....

# 7. 文本编辑器编写程序

  1. 掌握C语言的语法规则.
  2. 电脑上安装C语言编译器

有了以上两者，我们就具备开发C语言程序的环境了. 只不过环境稍微简陋一些.

  1. 创建C语言的程序文件. 这个文件的扩展名必须是 .c
  2. 在 .c 文件中编写我们的第一个程序.
  3. 使用编译器手动将我们编写的源代码转换成可执行程序.

缺点：

如果编写一个程序文件，并且程序文件的代码还不多，那么使用文本编辑器编写那就无所谓. 但是如果代码比较多、程序文件也比较多。

所以，我们真正去开发一个C语言项目的时候，不会去使用 文本编辑器，而是使用集成开发环境 IDE。

  1. 帮我们管理目录结构.
  2. 能够帮助我们进行调试，帮助我们非常容易的定位错误.
  3. 当我们编写程序的时候，能够即时提示程序中的错误.
  4. 语法高亮.
  5. 编译运行代码的时候，一键完成. 而不是在命令行自己敲命令.
  6. 等等....

专业的软件开发工具.

编译步骤:

  1. 开始找到 Developer Command Prompt for VS 2019 或者 Developer PowerShell for VS 2019 并且点击打开.
  2. 将 test.c 放在桌面上
  3. 在命令行中输入: cd 跟上test.c所在的目录
  4. 在命令行中输入: dir 命令，查看当前目录下有哪些文件
  5. 在命令行中输入: cl test.c 在桌面上生成两个文件
    1. xxx.obj
    2. xxx.exe
  6. 在命令行继续输入: .\ xxx.exe 看能够输出 hello world.

# 9. VS2019 使用和配置

  1. 项目创建.

    创建控制台的空项目.

  2. 字体、风格配置. 工具 -> 选项 -> 环境 -> 字体和颜色

  3. 掌握一些常用的快捷键.

    1. F5 调试运行程序、CTRL + F5 运行程序不调试.
    2. 代码编辑区放大： CTRL + SHIFT + >、 放小： CTRL +SHIFT + <
    3. 块选择： ALT + 鼠标左键
    4. 调到指定的行号: CTRL + G

  4. 掌握程序的编译和运行、运行几种模式.

    ​ 运行代码有四种模式:

    1. Debug x86, 以调试的方式运行32位程序.
    2. Debug x64, 以调试的方式运行64位程序.
    3. Release x86, 以发布模式运行32位程序.
    4. Release x64. 以发布模式运行64位程序.
    5. Debug ， 在开发阶段一般都使用调试模式，当程序出现错误的时候，能够准确定位程序错误位置. 当使用 Debug 模式编译代码的时候，生成的 exe 程序就会比较大. 因为包含程序调试信息.
    6. Release，部署阶段. 生成的 exe 程序中不包含调试相关的信息. exe 程序比较小.

  5. 第一个 Hello World 程序.

    1. 当我们点击运行的时候，其实做了两件事情：
      1. cl xxxx.c, 生成了 xxxx.exe 程序， exe 叫做可执行程序.
      2. xxxx.exe 直接运行.

    工具 -> 代码片段管理器 -> 在语言这一栏选择 Visual C++ -> 添加

# 12. system("pause") 作用

  1. system("pause") 当指定这行代码的时候，程序就会暂停.
  2. system() 函数(功能)， "pause" 叫做功能的参数.
  3. pause 本质上是一个系统的终端命令. 就是通过程序的方式指定终端命令.
  4. 接触过的终端命令:
    1. cd 切换目录 change(改变) directory(目录)
    2. dir 列出当前目录下的所有文件和文件夹
    3. cls 清屏 clear。 system("cls")
    4. pause 暂停
  5. F5 运行完毕如果没有碰到暂停，会直接关闭窗口. CTRL + F5 运行程序，此时不会关闭窗口.

# 13. 注释的作用

比如：现在开发好程序，但是过了半年. 拿到需求，让我们要改动我们编写的代码. 此时，打开半年前写的代码. 你可能会说一句: 我靠，这是我写的代码吗？这是啥意思？为什么这么写？有什么作用？
